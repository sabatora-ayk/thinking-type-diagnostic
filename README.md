# 思考タイプ診断（Thinking Type Diagnostic）

> あなたは「どんな人」ではなく、「どう考える人」なのか。

ブラウザだけで3〜5分、16問の質問に答えると、今回の回答から見えた
「思考の傾向」を4つの軸で示す診断です。

## Demo

https://thinking-type-diagnostic.vercel.app

## About

4つの認知傾向から、自分の思考スタイルを探索する実験的診断ツールです。
---

## この診断について、最初にお伝えしたいこと

本診断は、いわゆる性格診断・エンタメ診断とは異なる方針で設計されています。

### 実験的な認知傾向モデルです

この診断は、確立された心理学的尺度ではありません。「情報をどう受け取り、
整理し、判断し、行動につなげるか」という**認知プロセス**を捉えようと
試みている、**検証段階の実験的なモデル**です。

設計にあたっては、既存の意思決定研究・認知心理学の知見（Recognition-
Primed Decision, Need for Cognitive Closure, Situated Action 等）を
参照していますが、これらの理論をそのまま採用しているわけではなく、
独自の仮説として構成し、その仮説が実データでどこまで成立するかを
今後検証していく段階にあります。

### 性格や能力を固定的に分類するものではありません

結果画面のタイプ名は、「あなたは◯◯という人間である」という断定では
なく、「今回の回答パターンでは、◯◯の傾向が強く出た」という一時点の
傾向を示すものです。同じ人でも、状況や気分によって回答が変わりうる
前提で設計されています。

また、質問の設計過程では「性格」「価値観」「道徳観」「経験」「能力」
「文化的背景」といった、認知プロセス以外の要素をできる限り測らない
ように、複数回にわたる自己レビュー（認知インタビュー・交絡因子の
検証）を重ねています。ただし、それでも完全に他の心理特性と切り離す
ことはできていない部分があることも、設計記録として正直に残しています
（`docs/` 以下のADR・設計ドキュメントを参照）。

## 現在の4軸モデル（v0.1）

| 軸 | 測っていること |
|---|---|
| **直感 ←→ 分析** | 新しい情報を理解するとき、過去の類似パターンとの照合で一気に意味を掴むか、要素を一つずつ確認して積み上げるか |
| **探索 ←→ 収束** | 複数の選択肢がある状況で、並行して試し続けるか、早めに一つへ絞り込むか |
| **実績 ←→ 規範** | 方法や判断の妥当性を、自分自身の実体験の結果で確かめるか、一般的に確立された基準で確かめるか |
| **事前構築 ←→ 相互構築** | 物事に取り組むとき、行為の前に理解を完成させておきたいか、行為をしながら理解を完成させていきたいか |

これら4軸の組み合わせにより、16のタイプ（例：冒険者、探究者、
戦略家など）のいずれかが結果として表示されます。

---

## v0.1という位置づけについて

このモデルは `v0.1` として公開されています。これは「今後変わる
可能性がある試作版」であることを意味します。

具体的には、今後以下のような検証・見直しを行う計画です。

- 内的一貫性（Cronbach's α）の検証
- 因子分析による4因子構造の再現性確認
- 軸同士の相関検証（特に「直感←→分析」と「事前構築←→相互構築」の
  弁別性は、設計段階から未解決の論点として記録されています）
- 既存の心理尺度（自己効力感、認知的完結欲求など）との判別的妥当性の検証

検証の結果によっては、軸の統合・再定義・設問の差し替えが行われる
ことがあります。設計の意思決定はすべて `docs/adr/` にADR
（Architecture Decision Record）として記録されており、
「なぜこの設計にしたか」「何を採用せず、何を採用したか」を
誰でも追跡できるようにしています。
